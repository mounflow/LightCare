## 0. 设计原则（贯穿全文）

| 原则 | 含义 | 反例 |
|---|---|---|
| **低负担记录** | 2 步内完成，拍照 / 语音 / 搜索三种"零思考"方式 | 强制选品牌、强制选重量 |
| **正向激励** | 永远先给肯定，再说"可以更好" | "今日超标 +320 kcal" |
| **家庭化温度** | 像家人/朋友的口吻，可被爸妈辈看懂 | 临床术语、英文术语 |
| **不构成医疗建议** | 任何异常提示都引导咨询医生 | 自行下诊断 |
| **可拒绝 / 可改** | 任何推荐都提供"稍后再说 / 换一换" | 强推方案 |

## 1. 产品定位与目标用户

### 1.1 一句话定位

> **"轻养"是一款为全家人设计的饮食与作息陪伴 APP——用 10 秒拍一餐的方式，帮你和家人慢慢吃得更均衡，告别热量焦虑。**
>
> **产品形态**：免费 + 云端账号。手机号/微信一键登录，数据云端同步，最多 4 份家庭档案共享一个账号，成员之间可远程互发鼓励徽章与查看亮点。

### 1.2 核心价值主张

1. **零压力记录**：拍照 / 语音 / 搜索 3 种方式，最快 5 秒完成一餐。
2. **均衡优先于卡路里**：主反馈指标是"蛋白质"和"蔬果达成率"，热量只做参考。
3. **家人可同屏**：一份家庭档案，多个成员独立记录、互相点亮"鼓励"小徽章。
4. **不内疚的提醒**：连续 2 天没记录不发红字，第 3 天起只发一句"今天也想跟你打招呼"。

### 1.3 典型用户画像

> 原则：**"所有人"都是潜在用户**——从 14 岁到 80 岁，从单身到多代同堂，从主动管理健康到被家人代为记录。
> 下面 4 个画像覆盖**不同使用动机与使用能力**，而不是覆盖所有人群。真实用户可以同时是其中 1-2 种。

#### 画像 A · 自我驱动型（25–45 岁，注重生活质量的成年人）
- 知道"应该吃得更均衡"，但卡路里计数 / 健身食谱坚持不下来。
- 不喜欢"被监督"的口吻，讨厌红字警告。
- 想要一种"轻、温柔、能看见自己进步"的工具。
- **核心场景**：早 7 点拍早餐、午间快速记一餐、睡前看一眼今日色环。

#### 画像 B · 家庭照顾者（30–50 岁，替家人代填的人）
- 给爸妈 / 伴侣 / 孩子单独建档，自己是"录入员"也是"关心者"。
- 希望不用切换账号就能在多份档案之间来回看。
- 喜欢鼓励徽章这种"小情感表达"。
- **核心场景**：晚饭后帮爸点开语音、给妈点一颗"真棒"。

#### 画像 C · 被照顾者（55 岁以上，主要被他人关怀的长辈）
- 不擅长输入文字，依赖语音 / 大按钮。
- 关注的是"今天菜够不够 / 步数到没到"，不需要看到克数。
- 对"失败""未达标"等字眼敏感，需要被温柔对待。
- **核心场景**：被家人拉着点开 APP，用长按语音记一餐；偶尔自己点开看看色环。

#### 画像 D · 成长中的孩子（14–18 岁，家长代为建立档案）
- 处于挑食 / 不规律饮食的年龄，自主性差。
- 家长需要"看见孩子吃了什么"，但又不能逼得太紧。
- **核心场景**：午餐后家长翻看孩子的色环；孩子用拍照/搜索快速打卡。

> **设计取舍**（基于"全家人"定位）：
> 1. **不预设家庭结构**——既支持"单身一人用"，也支持"一家四口共用一个 APP"。建档时不强制填关系。
> 2. **多档案共屏**——底栏"我的"区永远可一键切换"我 / 其他成员"，最多支持 4 份档案。
> 3. **使用能力自适应**——根据档案的"年龄 + 关系"自动调整字号、按钮大小、是否默认显示数字、是否开启语音优先。
> 4. **隐私边界**——跨档案可见的只有"亮点 / 鼓励徽章"，具体克数 / 体重等只对本人与"主档案"可见（家长可设为"我作为孩子档案的代填人"）。

### 1.4 营养目标默认值（按年龄段 3 档）

> 不在首屏展示，藏在"我的 → 高级目标"里。建档时按出生年自动套用，事后可逐项微调。

| 指标 | 14-18 岁 | 19-50 岁 | 51+ 岁 |
|---|---|---|---|
| 蛋白质（g/日） | 60 | 60 | 55 |
| 蔬菜（份/日，1 份 ≈ 80 g） | 5 | 5 | 5 |
| 水分（ml/日） | 1500 | 1700 | 1500 |
| 步数（步/日） | 8000 | 8000 | 6000 |
| 热量（kcal/日，仅参考不主推） | 2000 | 2000 | 1800 |

> 体质差异较大（体力劳动者、孕期、慢病）由用户在"高级目标"中微调；APP 不做自动判断。

## 3. 核心用户操作流程

> 流程图用文字描述，对应 UI 详见 §4。

### 3.1 记录一餐（Happy Path：拍照）

```
[1] 任意 Tab 点中央大号 FAB "+"
        │
        ▼
[2] 进入拍照页（默认全屏相机）
   - 框中餐食 → 点底部白色圆形快门
        │
        ▼
[3] 0.8 s 后 AI 识别 → 弹出识别结果浮层
   - 识别到 N 个食材（带绿点 ✓）
   - 未识别食材显示"?"让用户补
   - 顶部三标签切换：早餐 / 午餐 / 晚餐 / 加餐（智能预选）
   - 食量三档：🍽小 / 适中 / 大（默认适中）
        │
        ▼
[4] （可选）点击"+ 文字说明"补一句话 → "妈妈做的红烧肉"
        │
        ▼
[5] 点底部"✓ 记一笔"
   - 触发"上滑"成功动画（一片小绿叶飘起）
   - 1.2 s 后自动返回首页 → 圆环进度自动 +1
```

**异常路径**：
- 拍糊了 / 没人 / 多张图 → 顶部 toast"没看清哦，要不要再拍一张？" + 兜底"用文字记"。
- 网络差 → 先存本地，状态栏小角标"待同步"。

### 3.2 首次建档（2 分钟内）

```
Splash → 步骤指示器（3 圆点）
  ├─ Step 1  欢迎语
  │          "我们相信健康是一段旅程，不是一场竞赛"
  │          大按钮 "开启健康旅程"（绿色 pill）
  │
  ├─ Step 2  建立你的档案
  │          - 头像（默认豆豆人，可拍 / 选系统）
  │          - 昵称
  │          - 性别 / 出生年（仅用于估算，不强制）
  │          - 关系（我 / 家人 A / 家人 B / 家人 C）—— 可选，不填也行
  │          大按钮 "继续"
  │
  ├─ Step 3  你的小目标
  │          - 大字号三选一卡片：
  │            "想吃得更均衡" / "多陪家人吃饭" / "不内疚地吃"
  │          - 选完即结束
  │          底部小字："随时可以在『我的』里改"
  │
  ▼
进入首页，圆环显示"0%"
```

> 关键设计：建档**不出现**身高 / 体重 / BMI 字段。这些是 P1 高级设置里再问，避免首屏吓退老人。

### 3.3 查看推荐并采纳

```
首页 "为您推荐" 区
  │
  ▼
卡片 1：饮食推荐（绿底）
  "今天蛋白质摄入不足！午餐加个金枪鱼三明治怎么样？清爽又助你轻松达标。"
  三个动作：[接受] [稍后再说] [换一换 →]
        │
        ├─ [接受] → 弹出"记为加餐" / "加入购物清单" / "查看做法" 三选项
        │           └─ 选"记为加餐" → 直接生成一条预填记录 → 确认即上滑成功
        │
        ├─ [稍后再说] → 卡片下沉，4 小时后再次上浮
        │
        └─ [换一换] → 横滑切到下一张（最多 3 张），动画轻快

卡片 2：运动推荐（白底 + 走路小人图标）
  "傍晚散散步？10 分钟的散步可以提升你的心情哦。"
  三个动作：[接受] [今天有点累，跳过] [换一换]
```

> "跳过" 按钮的颜色是**柔珊瑚色**而非红色，文案不带"不达标"字样。

## 4. 关键界面与交互说明

> 对应设计稿：`/stitch_greeneats/_1`（控制台）、`_2`（拍照）、`_3`（语音）、`_4`（周报）、`_5`（我的）、`_6`（欢迎页）。
> 间距基准：8 px 栅格，移动端外边距 20 px；卡片圆角 ≥ 24 px；按钮全部 pill 形。

### 4.1 首页 · 控制台

**布局**（自上而下）：

| 区域 | 元素 | 交互 |
|---|---|---|
| 状态栏 | 头像 + "轻养" / 右侧铃铛 | 点头像切换家庭成员；点铃铛进通知中心 |
| 问候 | 巨号 "早安！"（28 px Quicksand 700）+ 副文 "今天也要好好吃饭" | 纯展示 |
| 大色环 | 三层同心环（蛋白 8 / 8 + 蔬果 5 / 5 + 水分 6 / 8） | 长按弹"今天怎么补？" |
| 快捷目标 | 横向滚动 3 张小卡：蔬果达标 / 保持水分 / 心情打卡 | 点一下完成"打卡"动画 |
| 为您推荐 | 饮食卡（绿底）+ 运动卡（白底） | 见 §3.3 |
| 底部 Tab | 首页 / 洞察 / **中央大+号 FAB** / 目标 / 我的 | 中央 FAB 永远触发"记录" |

**关键交互细节**：
- 色环 stroke 粗 12 px，"未达成"轨道 10 % 透明度；达成时整环发光 0.6 s。
- 每次成功记录：对应色环 fill 动画 600 ms 缓动到新值，同时有"叮"短音效（可关）。
- 卡片悬浮：阴影从 5 % 提升到 10 % 模糊 + Y 轴 -2 px（不放大）。
- 切换家庭成员：色环、文案、推荐**全部刷新**，但底栏 Tab 不变。

### 4.2 记录页（3 种模式）

**入口**：从 FAB 弹出 Bottom Sheet，三选一（参考 `_2` 顶部的 `相机 / 语音 / 搜索` 横排）：

#### 4.2.1 相机模式（默认）

- 全屏相机预览，底部居中一个 **96 px 白色圆形快门**，外圈深绿 2 px 描边。
- 顶部右上 ✕ 关闭，顶部左上 闪光灯 / 相册切换。
- 快门上滑 = 视频，长按 = 多拍连拍（MVP 不做，仅长按拍照）。
- 拍完 0.5 s 内进 AI 识别 → 弹识别结果浮层（卡片在底部 1/3 高度，可上推全屏编辑）：
  - **食材列表**：每行 `✓ 绿点 + 食材名 + 份数（自动）`，点 ✓ 取消 / 点食材名编辑。
  - 顶部 chip：`早 / 午 / 晚 / 加餐`（默认根据当前时间推断）。
  - 食量：`小 / 适中 / 大`（默认适中）。
  - 备注区：可折叠 "+ 加一句"。
  - 底部按钮：`✓ 记一笔`（pill 绿色）。
- 识别失败：浮层变成"没看清，要不要试语音或搜索？"+ 两个备选按钮。

#### 4.2.2 语音模式（`_3` 已实现）

- 中央一个 **120 px 深绿圆**，外圈柔光晕，深绿麦克风图标。
- 文字泡显示实时识别的中文（一行，placeholder："中午吃了一碗牛肉面，一盘青菜…"）。
- 底部大 pill 按钮：**长按录音**（按住 = 录，松手 = 停 + 自动分析）。
- 底部三 tab 切换模式：相机 / 语音（当前）/ 搜索。
- 录音停止后：自动分析为 1-N 条记录预览，用户点 ✓ 即批量写入。
- **首版仅支持普通话**（用系统 Speech API）。识别失败或无网络 → 浮窗兜底"没听清，要不要试搜索？"

#### 4.2.3 搜索模式

- 顶部搜索框（pill 形），placeholder："搜食物，如'番茄炒蛋'"。
- 联想词：常吃（基于历史 Top 5）+ 食材库分类（蔬果 / 主食 / 蛋白 / 饮品）。
- 每条结果卡片显示：图标 + 名称 + "约 1 份" + 营养素小条。
- 选完一项后自动回到搜索框，可继续加多选；右下角浮出 **"记 N 笔"** 浮动按钮。

### 4.3 推荐卡片（详见 §3.3）

**视觉规范**（与设计稿 `_1` 一致）：
- 饮食卡：背景 `secondary-container`（薄荷绿），左侧大圆角图标 64 × 64，主标题 Quicksand 24 px / 600，副文 16 px / 400。

## 5. 智能推荐规则引擎

### 5.1 饮食推荐：基于"过去 3 天营养素缺口"

**输入**：
```
{
  nutrient_gap: { protein_g: 18, veg_servings: 2, water_ml: 350 },
  meal_time: "lunch" | "dinner" | "snack",
  time_of_day: 13,
  last_meal_logged_min: 165,
  user_prefs: { dislikes: ["香菜"], allergies: ["虾"] }
}
```

**决策伪代码**：

```python
def recommend_meal(input):
    # 1) 缺口分级
    if input.nutrient_gap.protein_g >= 15 and input.nutrient_gap.veg_servings >= 2:
        intent = "balanced_combo"        # 蛋白+蔬果双缺口
        priority = "P0"
    elif input.nutrient_gap.protein_g >= 15:
        intent = "protein_focus"
        priority = "P0"
    elif input.nutrient_gap.veg_servings >= 2:
        intent = "veg_focus"
        priority = "P1"
    elif input.nutrient_gap.water_ml >= 400:
        intent = "hydration"
        priority = "P2"
    else:
        return None                    # 没缺口就不推（避免打扰）

    # 2) 菜谱池过滤
    pool = MEAL_POOL[intent]                           # 每个 intent 至少 20 个候选
    pool = [m for m in pool if not m.has(input.user_prefs.dislikes)]
    pool = [m for m in pool if not m.contains(input.user_prefs.allergies)]

    # 3) 个性化
    pool = sorted(pool, key=lambda m: m.user_history_score, reverse=True)
    top3 = pool[:3]

    # 4) 文案生成（模板 + 缺口数字）
    title  = TITLE_TEMPLATES[intent].format(gap=gap_text(intent, input.nutrient_gap))
    body   = BODY_TEMPLATES[intent].pick(pool)        # 跟菜相关
    return Card(title=title, body=body, meals=top3, priority=priority)
```

**文案模板示例**（保持朋友式口吻）：

| Intent | 标题 | 副文 |
|---|---|---|
| `balanced_combo` | "蛋白质和蔬菜都还差点～" | "午餐加个金枪鱼三明治怎么样？清爽又助你轻松达标。" |
| `protein_focus` | "今天蛋白质摄入有点少哦" | "晚餐来份豆腐蒸虾仁？简单又补蛋白。" |
| `veg_focus` | "蔬菜好像还没吃够呢" | "晚饭加一盘清炒西兰花？五颜六色心情也好。" |
| `hydration` | "今天喝的水不太够" | "下午泡杯花茶，暖暖的也补水。" |

**触发时机**：
- 午餐推荐：11:30 推送（用户可关闭）。
- 晚餐推荐：17:00 推送。
- 兜底：登录后首页"为您推荐"区永久保留 1 张最新推荐卡。

**反疲劳规则**：
- 同一张卡 24 h 内不再推第二次。
- 7 天内同一 intent 至少换 3 种不同菜谱。
- 用户点"稍后"或"换一换"超过 3 次 → 本周不再推同 intent。

### 5.2 运动推荐：基于"疲劳度 + 热量盈余 + 运动历史"

**输入**：
```
{
  fatigue_input: 0|1|2|3,        # 用户自报 0=精力充沛 3=很累
  calorie_surplus_kcal: 420,     # 今日摄入 - 今日目标 (摄入 - 消耗)
  steps_today: 4800,
  last_3_days_mins: [20, 0, 35], # 历史运动分钟
  time_of_day: 19,
  recovery_needed: false
}
```

**决策表**（先看疲劳度，再看盈余）：

| 疲劳度 | 热量盈余 | 步数 | 推荐 | 强度 |
|---|---|---|---|---|
| 3（很累） | 任意 | 任意 | **不推荐运动**，推送"今天好好休息"卡片 | — |
| 2 | < 200 | < 6000 | 5 分钟拉伸 | 极轻 |
| 2 | ≥ 200 | 任意 | 10 分钟舒缓散步 | 轻 |
| 1 | < 200 | < 6000 | 15 分钟散步 | 轻 |
| 1 | ≥ 200 | 任意 | 20 分钟快走 | 中 |
| 0 | < 200 | 任意 | 15 分钟散步 | 轻 |
| 0 | ≥ 200 | < 4000 | 25 分钟快走 | 中 |
| 0 | ≥ 200 | ≥ 4000 | 30 分钟慢跑 | 中-高 |

**决策伪代码**：

```python
def recommend_exercise(input):
    if input.fatigue_input == 3:
        return Card(
            title="今天好好休息吧",
            body="辛苦啦，一杯温水 + 早点睡，明天再动起来。",
            cta="记录睡眠",
            priority="info"
        )

    surplus = input.calorie_surplus_kcal
    steps   = input.steps_today
    fatigue = input.fatigue_input

    # 优先级：盈余 > 步数 > 疲劳
    if surplus >= 200 and steps < 4000 and fatigue == 0:
        plan = Plan(duration_min=30, type="慢跑", intensity="中-高")
    elif surplus >= 200 and fatigue <= 1:
        plan = Plan(duration_min=20, type="快走", intensity="中")
    elif fatigue == 2 and surplus < 200:
        plan = Plan(duration_min=5, type="拉伸", intensity="极轻")
    elif steps < 6000:
        plan = Plan(duration_min=15, type="散步", intensity="轻")
    else:
        return None                                  # 已达标不推

    return Card(
        title=f"傍晚{plan.type}？",
        body=f"{plan.duration_min} 分钟的{plan.type}可以提升你的心情哦。",
        cta="开始计时",
        skip_text="今天有点累，跳过"
    )
```

**运动完成回写**：
- 用户点"开始计时" → 跳计时页，到点震动提醒。
- 用户主动标记"完成 / 跳过"。
- 完成后回写 `last_3_days_mins`，影响明日推荐。

## 6. 非功能需求与设计原则

### 6.1 情感化设计

#### 6.1.1 文案风格（朋友式）

| 场景 | ❌ 不要 | ✅ 这样做 |
|---|---|---|
| 鼓励 | "今日摄入超标" | "今天吃得有点丰盛，明天回到节奏就好" |
| 缺口 | "蛋白质未达标 -18 g" | "蛋白质好像还差点，午餐加个蛋？" |
| 久未记录 | "您已 3 天未记录" | "今天也想跟你打个招呼 :)" |
| 体重波动 | "体重上升 0.8 kg" | "数字会上下波动，最近的均值挺稳的" |
| 推荐被拒 | "为什么拒绝建议" | "记下啦，下次换个更合口味的" |
| 周报 | "你失败了 N 项" | "这周你已经做对了 N 件事" |

#### 6.1.2 色彩使用

- **背景**：`#f8faf9`（Mist 雾白），避免纯白 #FFF 刺激眼睛。
- **主色**：`#16342d`（Deep Forest 深森绿），用于主按钮与主标题。
- **辅色 1**：`#32694e`（Sage 鼠尾草）/ `#b5efce`（薄荷绿），用于推荐卡背景。
- **辅色 2**：`#ff9782`（Coral 珊瑚），仅用于"跳过 / 警告"等需要温和注意力的场景，**绝不**用纯红。
- **水分蓝**：`#7eb6d4`（柔蓝），仅出现在"水"相关元素。
- **饱和度规则**：所有彩色保持低饱和（< 50 %），避免在晚间刺激。

#### 6.1.3 图标风格

- Material Symbols Rounded，字重 400，圆角端点。
- 一律 2 px 描边，无尖锐 90° 角。
- 推荐卡图标用 64 × 64 圆形薄荷绿底 + 32 px 内描线图标。

### 6.2 防放弃 / 留存机制

| 场景 | 触发条件 | 处理方式 |
|---|---|---|
| **1 天未记录** | 21:00 仍未点开 APP | 极轻推送："今天也见到你啦"（不点不催） |
| **2 天未记录** | 连续 2 天空 | 推送："想换个口味吗？试试" + 一张旧推荐卡 |
| **3 天以上未记录** | 连续 3 天 | 推送："没关系，我们慢慢来" + 简化记录入口（语音一键） |
| **体重短期波动 ≥ 1 kg** | 7 日滑动窗 | 周报内不展示数字；只显示"最近均值挺稳的" |
| **推荐被连续拒绝** | 同一 intent 被跳过 3 次 | 本周静默此 intent，改推次优先级 |
| **极端饮食**（如连续 3 天无蔬果） | 数据触发 | 不发"你不好"的文案，仅推"蔬菜小卡"（贴士型） |
| **家庭成员沉默 7 天** | 成员最后登录 > 7 天 | 主用户收到"是否给爸/妈打个电话？"小卡（不催促 APP 本身） |

**核心原则**：**永远不展示"失败"和"负向差值"**。数据只用来生成正向文案与推荐，不展示给用户看。

### 6.3 隐私与法律提示

1. **启动时一次性同意**（P0）：
   - 用户协议
   - 隐私政策（明确数据存哪里、是否上传云端）
   - **医疗免责声明**："本 APP 提倡生活方式调养，不作为医疗建议。如有具体健康问题，请咨询专业医疗人员。"
2. **设置页底部常驻** 12 px 灰字免责声明（见 §4.5）。
3. **数据最小化**：首版不要求身高体重；如用户后续在"高级目标"中填入体重，**仅本地加密存储**，不同步云端。
4. **可导出 / 注销**：设置 → 隐私 → 一键导出全部 JSON / 一键注销账号（30 天冷静期后真删）。
5. **未成年保护**：未满 14 岁禁用，14–18 岁需监护人邮箱验证；监护人可被指定为该档案的"代填人"。
6. **代填人数据边界**：代填人可看到被代填档案的"色环 %、亮点、鼓励徽章"，**看不到**：具体克数、体重、详细时间戳、备注。代填关系在"我的 → 家庭成员 → 编辑"中可随时解除。
7. **跨档案可见性**：其他家庭成员的档案默认对当前用户**完全不可见**；仅在主动"切换档案"时显示。
8. **食物识别数据**：照片只用于本地识别（ONNX 模型），原图默认不上传；用户可手动开启"上传以改进识别"。
9. **第三方 SDK 披露**：列在隐私政策中（含所用 OCR / 语音识别 SDK）。

### 6.4 性能与可访问性

| 项 | 目标 |
|---|---|
| 首屏可交互 | < 1.5 s（4G） |
| 拍照 → 识别 → 入库 | < 2.0 s（95 分位） |
| 推荐刷新 | < 300 ms |
| 字号最低 | 16 px（避免老人误触缩放） |
| 字号上限 | 跟随系统"超大文字"设置自适应（iOS Dynamic Type / Android FontScale 1.3-1.5） |
| 触控目标 | ≥ 48 × 48 dp |
| 色彩对比度 | ≥ WCAG AA（正文 4.5 : 1） |
| 深色模式 | 沿用 Mist 调，不强反白 |
| 离线 | 最近 3 天可读写 |
